AIエージェントのタスク完了品質を評価します。

## 使い方

```
/judge <bench_result_dir>
```

引数なしの場合は最新の bench 結果ディレクトリを使います。

## 手順

以下を順番に実行してください：

1. `$ARGUMENTS` が指定されていれば `test_results/$ARGUMENTS/results.txt` を、
   なければ `test_results/` 配下で最新のディレクトリの `results.txt` を読む

2. results.txt からタスク一覧（タスク文・結果・所要時間）を抽出する

3. 対応する `metrics_snapshot.jsonl` を読み、各タスクの `session_id` を特定する

4. 各タスクについて以下を収集する：
   - タスクのプロンプト
   - エージェントの最終回答テキスト（results.txt の「結果:」行）
   - `/data` ディレクトリのファイル一覧（docker exec langchain_app find /data -type f）
   - 要求されたファイルの内容（docker exec langchain_app cat <path>）
   - 必要に応じてコードを実行して動作確認（docker exec langchain_app python3 <path>）

5. 各タスクを以下の基準で **0〜5点** で採点する：

   | 点数 | 基準 |
   |------|------|
   | 5 | 完全に正しく完了。要求されたファイルが存在し、内容・実行結果が全て正確 |
   | 4 | ほぼ完了。軽微な問題（出力形式のずれ、コメント不足など）はあるが実質的に正しい |
   | 3 | 主要な目標は達成。一部のステップが欠けているか不正確 |
   | 2 | 部分的に完了。複数のステップが欠けているか、重要な要件を満たしていない |
   | 1 | ほとんど未完了。わずかな進捗のみ（ファイル作成のみ、実行なし など） |
   | 0 | 未完了。タスクを実行できていない、または空回答 |

6. 以下の形式でレポートを組み立てる：

```
## Judge レポート — <bench_result_dir>
モード: <react|plan_exec>  モデル: <model>  日時: <timestamp>

### タスク1: <プロンプト冒頭40文字>...
- ファイル存在: ✅/❌ <path>
- ファイル内容: ✅/❌/△ <コメント>
- コード実行:   ✅/❌/- <exit_code or N/A>
- 最終回答:     ✅/❌/△ <コメント>
**スコア: N/5** — <理由を1〜2文>

### タスク2: ...
（同様）

### タスク3: ...
（同様）

---
**合計: N/15**（平均 N.N/5）
所要時間との比較: <品質/時間のトレードオフを1文でコメント>
```

7. 組み立てたレポートを `test_results/<bench_result_dir>/judge_report.md` に Write ツールで保存する

8. 保存完了後、レポート全文をチャットにも出力する
