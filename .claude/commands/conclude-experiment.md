実験の考察をまとめ、総括ドキュメントを更新します。

## 手順

1. `$ARGUMENTS` で指定されたファイル名を使用する。
   未指定の場合は `docs/experiments/` 配下で最新のファイルを使用する

2. 実験ドキュメントを Read ツールで読む（目的・仮説・結果をすべて把握する）

3. `docs/experiments.md` を Read ツールで読む（既存の総括テーブルを確認する）

4. 実験ドキュメントの `## 考察` セクションを以下の観点で **追記** する：

   - **仮説の検証**: 仮説は正しかったか、なぜか
   - **主な発見**: 数値で示せる改善・悪化の事実
   - **副作用・注意点**: 改善した指標と引き換えに悪化した指標
   - **結論**: この実験設定を採用すべきか（採用 / 却下 / 条件付き採用）
   - **次のアクション**: この結果を踏まえた次の実験候補

   形式例：
   ```
   ## 考察

   ### 仮説の検証
   <検証結果>

   ### 主な発見
   - <発見1>
   - <発見2>

   ### 副作用
   - <副作用>

   ### 結論
   **<採用 / 却下 / 条件付き採用>** — <理由1〜2文>

   ### 次のアクション
   - [ ] <次の実験候補>
   ```

5. **ステータスを更新**する：`ステータス: 結果記録済み` → `ステータス: 完了`

6. `docs/experiments.md` の **全実験総括** テーブルに1行追加する：
   ```
   | 実験⑬ <name> | YYYY-MM-DD | <変更内容> | <14b主要指標> | <7b主要指標> | <採用/却下> |
   ```
   ※ テーブルが存在しない場合は新しいセクションを追記する

7. 以下を実行する：
   ```
   git add docs/experiments/<filename>.md docs/experiments.md
   git commit -m "experiment: conclude <name>"
   git push
   ```

8. 完了後に結果サマリーを1〜3文で出力する
